# fable-thinking

> **Операционная дисциплина Claude Fable 5 — как Claude Code скилл для Opus, Sonnet и Haiku.**

[English README](README.md) — основная версия. Здесь — русское описание; полный русский вариант скилла лежит в [`ru/`](ru/).

## Кому это нужно

Если вы замечали, что младшие модели *работают* не так, как топовая: говорят «готово» без проверки, патчат симптом вместо причины, теряют цель на десятом tool call. А гонять топовую модель на всём подряд — дорого: токены она потребляет очень быстро.

Этот скилл — ровно про этот компромисс. Он переносит **рабочую дисциплину** Fable 5 на модели уровнем ниже — чтобы Opus, Sonnet и Haiku делали ту же повседневную работу с дисциплиной топ-модели, за долю её токен-стоимости.

## Установка за 5 секунд

```bash
git clone https://github.com/uladzemer/fable-thinking ~/.claude/skills/fable-thinking
```

Claude Code подхватит скилл автоматически на нетривиальных задачах — или по запросу: *«думай как Fable»*, *«fable-mode»*. Русская версия скилла ставится копированием `ru/` в `~/.claude/skills/fable-thinking-ru/`.

## Что он делает

Промпт не делает модель умнее — веса не меняются. Но скилл переносит **дисциплину**: большинство провалов агентных задач — не провалы интеллекта, а провалы дисциплины. Скилл — это 8 процедур + 2 протокола, которые убивают классические провалы:

| Знакомый провал | Что делает скилл |
|---|---|
| «Готово!» — а проверки не было | **ДОКАЗАТЕЛЬСТВО**: никаких заявлений о завершении без запуска проверки и чтения вывода |
| Патч на патче, каждый «почти сработал» | **ДВА УДАРА**: 2 неудачные попытки по одной гипотезе → стоп, root-cause протокол |
| Починил одну копию задублированного кода | **ЧТЕНИЕ**: grep близнецов и call sites до первой правки |
| Решил не ту задачу, что просили | **ЦЕЛЬ**: одна проверяемая строка «готово, когда…», пере-заякоривание в середине |
| Молча угадал вместо «спросить» | **КАЛИБРОВКА**: каждое утверждение — Проверено / Выведено / Предположено |
| «Улучшил» то, о чём не просили | **ШАГИ**: стоп по достижении цели; границы scope |

Правило, определившее каждую строку скилла: «подумай ещё раз» без нового внешнего свидетельства делает модели **хуже** (Huang et al., ICLR 2024) — поэтому каждая самопроверка заякорена на внешний сигнал: команду, вывод, реально прочитанный код. Именно такой scaffolding даёт средним моделям двузначный прирост на агентных бенчмарках (Shinn et al., 2023).

## Когда НЕ использовать

- **Тривиальные правки** (rename, опечатка, one-liner) — скилл сам так говорит: церемония калибруется по размеру задачи.
- **На самом Fable 5.** Официальный гайд прямо предупреждает: скиллы для прошлых моделей «часто слишком прескриптивны для Fable 5 и могут ухудшать качество». Этот скилл — для моделей уровнем ниже.

Это перенос дисциплины, не способностей: знаний не добавит, горизонт планирования не удлинит. Максимальная отдача — на длинных, грязных, неоднозначных задачах.

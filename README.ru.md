# fable-thinking

> **Claude Fable 5 записал, как он думает, — чтобы Opus, Sonnet и Haiku могли одолжить эту дисциплину.**

[English README](README.md) — основная версия. Здесь — русское описание; полный русский вариант скилла лежит в [`ru/`](ru/).

**Установка за 5 секунд:**

```bash
git clone https://github.com/uladzemer/fable-thinking ~/.claude/skills/fable-thinking
```

Claude Code подхватит скилл автоматически. Скажи *«думай как Fable»* или просто начни нетривиальную задачу на Opus/Sonnet — скилл загрузится сам по описанию. Русская версия скилла ставится копированием `ru/` в `~/.claude/skills/fable-thinking-ru/`.

## История

Пользователь попросил Claude Fable 5 (фронтир-модель Anthropic класса Mythos): *«сделай скилл, который научит Opus 4.8 и Sonnet 5 думать как ты»*. Fable 5 подошёл к задаче честно:

1. **Проверил слух.** Официального механизма «передачи мышления Fable» младшим моделям **не существует** (проверено по анонсам и докам Anthropic; вероятный источник слуха — refusal-fallback: запросы, которые Fable 5 отклоняет, обслуживает Opus 4.8).
2. **Нашёл, что переносится на самом деле** — *операционная дисциплина*. Официальный [гайд Prompting Claude Fable 5](https://platform.claude.com/docs/en/build-with-claude/prompt-engineering/prompting-claude-fable-5) — фактически каталог поведений Fable 5, включая приём с измеренным эффектом: аудит статус-заявлений против tool results *«практически устранил сфабрикованные отчёты»* в тестах Anthropic.
3. **Дистиллировал собственные паттерны** в 8 процедур + 2 протокола (дебаг с нуля, само-ревью диффа).
4. **Прогнал A/B-тест** на Sonnet 5 сабагентах со скриптовым грейдингом.
5. **Выдержал пять адверсариальных ревью** — fresh-context самокритика, security-пас, прочтение «глазами Sonnet», кросс-модельные ревью **GPT-5.5 (Codex) и Gemini 3.5**. Все подтверждённые находки исправлены — включая YAML-баг, из-за которого скилл вообще молча не загрузился бы.

## Что он делает (честная часть)

Промпт не делает модель умнее — веса не меняются. Но скилл переносит **дисциплину**: большинство провалов агентных задач — не провалы интеллекта, а провалы дисциплины. Скилл убивает классические:

| Знакомый провал | Что делает скилл |
|---|---|
| «Готово!» — а проверки не было | **ДОКАЗАТЕЛЬСТВО**: никаких заявлений о завершении без запуска проверки и чтения вывода |
| Патч на патче, каждый «почти сработал» | **ДВА УДАРА**: 2 неудачные попытки по одной гипотезе → стоп, root-cause протокол |
| Починил одну копию задублированного кода | **ЧТЕНИЕ**: grep близнецов и call sites до первой правки |
| Решил не ту задачу, что просили | **ЦЕЛЬ**: одна проверяемая строка «готово, когда…», пере-заякоривание в середине |
| Молча угадал вместо «спросить» | **КАЛИБРОВКА**: каждое утверждение — Проверено / Выведено / Предположено |
| «Улучшил» то, о чём не просили | **ШАГИ**: стоп по достижении цели; границы scope |

Научная база: Reflexion поднимает HumanEval с 80% до 91% на той же модели; GPT-3.5 в агентном цикле (95.1%) обгоняет GPT-4 zero-shot (67%). И ключевая оговорка, определившая каждую строку скилла: **«подумай ещё раз» без нового внешнего свидетельства делает модели *хуже*** (Huang et al., ICLR 2024) — поэтому каждая самопроверка заякорена на команду, вывод или прочитанный код.

## Когда НЕ использовать

- **Тривиальные правки** (rename, опечатка, one-liner) — скилл сам так говорит: церемония калибруется по размеру задачи.
- **На самом Fable 5.** Официальный гайд прямо предупреждает: скиллы для прошлых моделей «часто слишком прескриптивны для Fable 5 и могут ухудшать качество». Этот скилл — для моделей уровнем ниже.

## Честные ограничения

- На коротких, чётко поставленных задачах Sonnet 5 и так дисциплинирован — наши A/B-прогоны насытились (обе руки ~10/10). Отдача — на **длинных, грязных, неоднозначных** задачах.
- Это перенос дисциплины, не способностей: знаний не добавит, горизонт планирования не удлинит.
